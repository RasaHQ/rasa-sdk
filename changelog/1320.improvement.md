**Barge-in / interruption support**

When the user speaks over the assistant (barge-in), the voice channel can notify the action server via `AckStreamChunks`. The action server stops producing chunks and records only the already-delivered chunks to the tracker.

New `CollectingDispatcher` API:

- `cancel_stream()` — signals that the user interrupted. Subsequent `stream_chunk()` calls are silently dropped. The action continues running so it can still return Rasa events (slot sets, etc.).
- `is_streaming_cancelled` property — `True` after `cancel_stream()` has been called. Action authors can check this to optionally suppress side effects after a barge-in.

New gRPC RPC:

- `AckStreamChunks(StreamChunkAck) returns (google.protobuf.Empty)` — the voice channel calls this with the `response_id` of the active streaming response. The action server looks up the dispatcher in its registry and sets the cancellation flag. `WebhookStream` detects the flag on the next iteration, drains the queue until the action finishes, and yields `final_result` (containing all Rasa events) to the client.

The `response_id` field on `ChunkStart`, `Chunk`, and `ChunkEnd` events acts as the correlation key between the outbound stream and the inbound `AckStreamChunks` call.
